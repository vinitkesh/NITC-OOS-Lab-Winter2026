## Plugin Architecture

Description: Plugin architecture allows for extending the functionality of an application without modifying its core code. This is often achieved through interfaces and dynamic loading of classes at runtime.

example: A simple example of a plugin architecture could be a text editor that supports various file formats through plugins. Each plugin implements a common interface for opening and saving files, allowing the editor to support new formats without changing its core code.

## Plugin Architecture in Java using interfaces and ServiceLoader

In Java, you can create a plugin architecture using interfaces and the `ServiceLoader` class. Here's a simple example to demonstrate this:

1. Define a plugin interface:

```java
public interface TextPlugin {
    String getPluginName();
    void performAction(String text);
}
```

2. Create a plugin implementation:

```java
public class UpperCasePlugin implements TextPlugin {
    @Override
    public String getPluginName() {
        return "UpperCase Plugin";
    }

    @Override
    public void performAction(String text) {
        System.out.println(text.toUpperCase());
    }
}
```

3. Create a service provider configuration file:
In the `META-INF/services` directory, create a file named `TextPlugin` with the following content:

What does `META-INF/services/TextPlugin` contain?
Ans: 
- The `META-INF/services/TextPlugin` file contains the fully qualified name of the plugin implementation class. In this case, it would contain: `UpperCasePlugin`
- META-INF is a special dir in java for jar files to store metadata.

4. Load and use the plugin in the main application:
```java
import java.util.ServiceLoader;

public class PluginDemo {
    public static void main(String[] args) {
        ServiceLoader<TextPlugin> loader = ServiceLoader.load(TextPlugin.class);
        for (TextPlugin plugin : loader) {
            System.out.println("Loaded plugin: " + plugin.getPluginName());
            plugin.performAction("Hello, World!");
        }
    }
}

```
